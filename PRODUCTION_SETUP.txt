==============================================================================
 AGENTIC SECURITY INTELLIGENCE — PRODUCTION SETUP GUIDE
==============================================================================
 Audience: an SRE/platform engineer standing this up from scratch.
 The offline/deterministic stubs were removed — every backend below is REQUIRED
 (except the ones explicitly marked optional). The app FAILS CLOSED at boot
 (ENVIRONMENT=prod) if anything mandatory is missing; it prints exactly what to fix.

 Read top to bottom. Bring up infra (sections 2–10), then the app (section 11),
 then validate (section 13).

------------------------------------------------------------------------------
 0. ARCHITECTURE AT A GLANCE
------------------------------------------------------------------------------
 Clients ──(X-API-Key + Bearer JWT)──> FastAPI app (this repo)
    app ──> Postgres + RLS        (chat sessions/messages; tenant isolation)
    app ──> Qdrant                (RAG vector store; READ-only — see §4)
    app ──> TEI embedder          (Qwen3-Embedding; query + doc vectors)
    app ──> TEI reranker          (optional; cross-encoder precision)
    app ──> SGLang LLM            (Qwen 7B/72B + DeepSeek; 3 lanes)
    app ──> Prompt Guard 2        (prompt-injection classifier)        [guardrail]
    app ──> Llama Guard 3         (content-safety classifier)          [guardrail]
    app ──> Presidio (in-process) (PII redaction)                      [guardrail]
    app ──> Redis                 (optional cache / rate-limit)
    app ──> Langfuse              (optional tracing)
    EXTERNAL CRON ──> Qdrant      (writes report documents; NOT this app — see §4)
    app ──> EASM/Brand/ACI MCP servers (optional; via MCP_URLS — see §12)

 GPUs: the LLM (SGLang) and the embedder/reranker (TEI) and the two guard models
 need GPUs. Sizing depends on the models you pick (see each section).

------------------------------------------------------------------------------
 1. PREREQUISITES
------------------------------------------------------------------------------
 - Linux hosts (containers/k8s recommended). Docker or containerd + NVIDIA runtime.
 - NVIDIA GPUs with recent drivers + CUDA for SGLang/TEI/guard models.
 - Python 3.11 for the app.
 - A secrets manager (Vault / cloud secret store). NEVER bake secrets into images.
 - Network policy: the app reaches every backend; backends are NOT exposed publicly.
 - TLS terminating reverse proxy / gateway in front of the app (nginx, Envoy, ALB).

------------------------------------------------------------------------------
 2. POSTGRES  (chat store + Row-Level Security)  — REQUIRED
------------------------------------------------------------------------------
 Role: stores sessions + messages. Tenant isolation is enforced at the DB via RLS
 (org_id), on top of the app-layer checks (defense in depth).

 2.1 Provision Postgres 15+ (managed RDS/Cloud SQL or self-hosted). Create the DB:
     CREATE DATABASE asi;
     CREATE ROLE asi_app LOGIN PASSWORD '<strong>';
     GRANT ALL PRIVILEGES ON DATABASE asi TO asi_app;

 2.2 Connection string -> DATABASE_URL:
     DATABASE_URL=postgresql://asi_app:<pw>@<host>:5432/asi

 2.3 Run migrations (creates tables + RLS policies). The app ships a migrator
     (installed with the app in §13):
     asi-migrate                 # idempotent; run on deploy
     # (entry point: app.core.db.migrate:main)

 2.4 RLS: the app sets a per-transaction GUC to the caller's org so Postgres
     enforces "an org sees only its own rows". The session variable name is
     RLS_SETTING_NAME (default: app.organization_id). The migrator wires the
     policies; do not disable RLS in prod.

 2.5 Connection pool: DB_POOL_MIN / DB_POOL_MAX (defaults 1/10). Size to
     (workers x per-worker concurrency). Put PgBouncer in front for many workers.

 2.6 Backups: automated PITR snapshots; test restores. This holds customer chats.

------------------------------------------------------------------------------
 3. QDRANT  (RAG vector store)  — REQUIRED
------------------------------------------------------------------------------
 Role: stores report-document embeddings; the agent searches it at query time.
 The app only READS Qdrant — documents are written by your external cron (§4).

 3.1 Run Qdrant (Docker):
     docker run -d --name qdrant -p 6333:6333 -p 6334:6334 \
       -v /data/qdrant:/qdrant/storage qdrant/qdrant:latest
     QDRANT_URL=http://qdrant:6333
     QDRANT_API_KEY=<key>        # set QDRANT__SERVICE__API_KEY on the server; required in prod

 3.2 Collection contract (the cron creates this; the app expects it to exist):
     - name: reports_kb  (matches REPORTS_COLLECTION)
     - vector size: EMBEDDING_DIM (default 1024) — MUST match the embedder (§5)
     - distance: Cosine
     - payload MUST include: org_id (string), text, source, doc_id, title,
       published_at (ISO-8601), section, metadata
     - create a payload index on org_id (keyword) for fast tenant-filtered search.

 3.3 Tenant isolation: every search the app issues is hard-filtered by org_id
     (mandatory `must` match), taken from the verified token — never the query.
     Therefore every point the cron writes MUST carry the correct org_id.

------------------------------------------------------------------------------
 4. THE REPORTS INGESTION CRON  (EXTERNAL — you operate it; NOT this app)
------------------------------------------------------------------------------
 This app does NOT ingest. Your cron embeds + upserts report documents into Qdrant.
 Contract it MUST honor so the agent can retrieve correctly:
   - Embed with the SAME model + dim as TEI (§5): EMBEDDING_MODEL / EMBEDDING_DIM.
   - Upsert into the SAME collection: reports_kb.
   - Stamp every point with org_id (the owning tenant) — this is the isolation key.
   - Chunk long documents; give each chunk a stable id (e.g. "<doc_id>::<n>") and
     a doc_id so re-ingest/delete can find all pieces.
   - Set published_at (ISO-8601) so recency re-weighting works.
 If the embedding model/dim drifts from the app's, retrieval silently degrades —
 keep them pinned to the same values.

------------------------------------------------------------------------------
 5. TEI EMBEDDER  (Qwen3-Embedding)  — REQUIRED
------------------------------------------------------------------------------
 Role: turns the question (and the cron turns documents) into vectors.

 5.1 Run Hugging Face Text Embeddings Inference (TEI) on a GPU:
     docker run -d --name tei-embed --gpus all -p 8080:80 \
       -v /data/hf:/data \
       ghcr.io/huggingface/text-embeddings-inference:latest \
       --model-id Qwen/Qwen3-Embedding-8B
     EMBEDDING_PROVIDER=tei
     TEI_EMBED_URL=http://tei-embed:8080
     EMBEDDING_MODEL=Qwen/Qwen3-Embedding-8B
     EMBEDDING_DIM=1024          # MUST equal the Qdrant collection vector size

 5.2 Alternative (no GPU / quick start): EMBEDDING_PROVIDER=openai with
     OPENAI_API_KEY + EMBEDDING_MODEL=text-embedding-3-large (set EMBEDDING_DIM to
     match). NOTE: this sends text to an external API — not for sensitive prod data.

------------------------------------------------------------------------------
 6. TEI RERANKER  (Qwen3-Reranker)  — OPTIONAL (quality boost)
------------------------------------------------------------------------------
     docker run -d --name tei-rerank --gpus all -p 8081:80 -v /data/hf:/data \
       ghcr.io/huggingface/text-embeddings-inference:latest \
       --model-id Qwen/Qwen3-Reranker-8B
     RERANK_ENABLED=true
     RERANK_PROVIDER=tei
     TEI_RERANK_URL=http://tei-rerank:8081
     RERANK_TOP_K=6

------------------------------------------------------------------------------
 7. LLM SERVER  (3 lanes)  — REQUIRED   [sglang | vllm | openai]
------------------------------------------------------------------------------
 Role: the reasoning model. Three "lanes": FAST (routing/planning/summary),
 STANDARD (answers), DEEP (hard multi-step/cross-domain synthesis). All three
 provider options speak the OpenAI-compatible HTTP API.

 7.1 Serve an OpenAI-compatible endpoint with SGLang (one model shown; you can run
     several SGLang instances, one per lane, or one endpoint serving multiple ids):
     python -m sglang.launch_server \
       --model-path Qwen/Qwen2.5-72B-Instruct \
       --host 0.0.0.0 --port 30000 --api-key <key>
     LLM_PROVIDER=sglang
     SGLANG_BASE_URL=http://sglang:30000/v1
     SGLANG_API_KEY=<key>
     MODEL_FAST=Qwen/Qwen2.5-7B-Instruct
     MODEL_STANDARD=Qwen/Qwen2.5-72B-Instruct
     MODEL_DEEP=deepseek-ai/DeepSeek-V3.1
     LLM_TEMPERATURE=0.1
     LLM_MAX_TOKENS=1024
     LLM_TIMEOUT_SECONDS=60

 7.2 The lane->model map (MODEL_FAST/STANDARD/DEEP) must name models the endpoint
     actually serves. If you only have one model, point all three at it (the DEEP
     lane simply collapses to it).

 7.3 vLLM (e.g. one 32B for STAGING — OpenAI-compatible; all 3 lanes collapse to the
     single served model):
     python -m vllm.entrypoints.openai.api_server \
       --model Qwen/Qwen2.5-32B-Instruct --host 0.0.0.0 --port 8000 --api-key <key>
     LLM_PROVIDER=vllm
     VLLM_BASE_URL=http://vllm:8000/v1
     VLLM_API_KEY=<key>
     VLLM_MODEL=Qwen/Qwen2.5-32B-Instruct

 7.4 OpenAI / hosted (quick start, no GPU): LLM_PROVIDER=openai + OPENAI_API_KEY +
     OPENAI_MODEL (all lanes collapse to one model). External API — not for sensitive
     prod data.

------------------------------------------------------------------------------
 8. PROMPT INJECTION GUARDRAIL — LLAMA PROMPT GUARD 2  — REQUIRED in prod
------------------------------------------------------------------------------
 Role: classifies the USER's question for prompt-injection / jailbreak; the app
 BLOCKS the turn on a positive. (The old hand-rolled regex screen was removed —
 this model IS the detector.)

 8.1 Serve meta-llama/Llama-Prompt-Guard-2-86M (or -22M) behind a small HTTP
     service exposing POST /classify that returns HF text-classification output,
     e.g. [{"label":"INJECTION"|"BENIGN","score":0.0-1.0}]. A minimal wrapper:
       - load the model with transformers `pipeline("text-classification", ...)`
       - POST /classify {"inputs": "<text>"} -> the pipeline output as JSON.
     PROMPT_GUARD_URL=http://prompt-guard:8085
     PROMPT_GUARD_THRESHOLD=0.8     # min malicious/jailbreak score to block

 8.2 The app calls:  POST {PROMPT_GUARD_URL}/classify  {"inputs": "<question>"}.
     A label in {injection, jailbreak, malicious, unsafe, label_1} at/above the
     threshold => BLOCK.

------------------------------------------------------------------------------
 9. CONTENT-SAFETY GUARDRAIL — LLAMA GUARD 3  — REQUIRED in prod
------------------------------------------------------------------------------
 Role: classifies the question for genuinely harmful content; BLOCKS on "unsafe".
 Tuned categories so legitimate offensive-security discussion is allowed.

 9.1 Serve meta-llama/Llama-Guard-3-8B as an OpenAI-compatible CHAT endpoint
     (SGLang/vLLM both work):
     python -m sglang.launch_server --model-path meta-llama/Llama-Guard-3-8B \
       --host 0.0.0.0 --port 8086
     LLAMA_GUARD_URL=http://llama-guard:8086/v1
     LLAMA_GUARD_MODEL=meta-llama/Llama-Guard-3-8B

 9.2 The app sends the text as a user turn to /chat/completions and BLOCKS when the
     reply starts with "unsafe".

------------------------------------------------------------------------------
 10. PII GUARDRAIL — MICROSOFT PRESIDIO  (in-process)  — REQUIRED when PII on
------------------------------------------------------------------------------
 Role: redacts PII from the OUTGOING answer (NER + context + checksums). Runs
 in-process in the app — no separate service. The app FAILS FAST at boot if it's
 enabled but not installed.

 10.1 Install with the app (the `pii` extra) + a spaCy model:
      pip install -e ".[prod]"            # prod includes presidio
      python -m spacy download en_core_web_lg
 10.2 Tuning (security product): PII_ENTITIES deliberately OMITS IP_ADDRESS/URL/
      DOMAIN (they are the subject matter, not PII). PII_SCORE_THRESHOLD=0.5.
      To disable entirely: PII_REDACTION=false.

 NOTE: secret redaction (API keys/JWT/PEM) and output-exfiltration defang and the
 RAG indirect-injection neutralization are CUSTOM (no library covers them) and are
 always on with GUARDRAILS_ENABLED=true — nothing to install.

 13. THE APP  (FastAPI)  — REQUIRED
------------------------------------------------------------------------------
 13.1 Install (ONE shot — prod includes Postgres/Qdrant/Redis/Langfuse/Presidio/
      FastMCP/LangGraph):
      python3.11 -m venv .venv && . .venv/bin/activate

 13.5 Capabilities: enable per deployment with CAP_*_ENABLED (reports/easm/brand/aci/

------------------------------------------------------------------------------
 14. AUTH  (API key gateway + JWT identity)
------------------------------------------------------------------------------
 Every request needs BOTH:
   - X-API-Key: a gateway key in API_KEYS (rotate via your secret store).
   - Authorization: Bearer <JWT> — identity (org_id, sub, roles) in the claims.
 The JWT is minted UPSTREAM (your gateway/IdP) and signed with JWT_SECRET (HS256);
 this app only verifies it. Requirements:
   - JWT_SECRET: 32+ random bytes, from the secret store (NOT the dev default).
   - API_KEYS: real key(s), comma-separated or JSON (NOT the dev default).
   - ACCESS_TOKEN_TTL_SECONDS: keep short (e.g. 3600).
 Upstream minting (reference): create the JWT with claims
   {sub, org_id, roles:[...], type:"access", iat, exp}  signed HS256 with JWT_SECRET.

------------------------------------------------------------------------------
 15. REMOTE MCP CAPABILITY SERVERS  (EASM / Brand / ACI / custom)  — OPTIONAL
------------------------------------------------------------------------------
 Promote a capability module to its own MCP server with ZERO code edits:
      MCP_URLS={"easm":"http://easm-mcp:8000/mcp","aci":"http://aci-mcp:8000/mcp"}
 The app mints a short-lived, org-scoped, audience-bound service token per call.
 Your MCP server MUST:
   - verify that Bearer token (audience = "<module>-mcp"; org from claims),
   - return tool results in {data, citations} shape (so the agent gets citations),
   - declare side-effecting tools so the human action gate applies.
 Enable the module's flag (e.g. CAP_EASM_ENABLED=true) alongside its URL.

 16. ENV VARS YOU ACTUALLY SET (prod) — quick reference
------------------------------------------------------------------------------
 Only the values below need setting; everything else has a default in config.py
 (override one by adding its UPPER_CASE name). This mirrors .env.example.
   ENVIRONMENT=prod
   API_KEYS=<real>   JWT_SECRET=<32+ bytes>   CORS_ORIGINS=<your origins>
   DATABASE_URL=postgresql://...
   QDRANT_URL=...    QDRANT_API_KEY=...    EMBEDDING_DIM=1024
   LLM_PROVIDER=sglang   SGLANG_BASE_URL=...  SGLANG_API_KEY=...
   EMBEDDING_PROVIDER=tei  TEI_EMBED_URL=...
   PROMPT_GUARD_URL=...  LLAMA_GUARD_URL=...
   TRACING_PROVIDER=langfuse  LANGFUSE_PUBLIC_KEY=...  LANGFUSE_SECRET_KEY=...   (recommended)
   MCP_URLS={...}  (if using remote modules)   CAP_*_ENABLED=...
 Sensible defaults you usually DON'T set: DEBUG=false, SEED_DEMO_DATA=false,
 RETRIEVAL_BACKEND=qdrant, STORE_BACKEND=postgres, RLS_SETTING_NAME=app.organization_id,
 ROUTER_MODE=llm, ORCHESTRATOR_MODE=planner, MAX_REPLANS=2, GUARDRAILS_ENABLED=true,
 GUARDRAILS_FAIL_CLOSED=true, PII_REDACTION=true, MODEL_FAST/STANDARD/DEEP, etc.

 17. PRODUCTION HARDENING CHECKLIST
------------------------------------------------------------------------------
 [ ] ENVIRONMENT=prod  (boot guard active; refuses insecure/incomplete config)
 [ ] DEBUG=false, SEED_DEMO_DATA=false
 [ ] JWT_SECRET + API_KEYS from a secret manager; rotated; not the dev defaults
 [ ] TLS everywhere; backends (Postgres/Qdrant/TEI/SGLang/guards/Redis) on a
     private network, never publicly reachable
 [ ] Postgres RLS enabled (asi-migrate ran); least-privilege DB role; PITR backups
 [ ] Qdrant API key set; org_id payload index present; every point carries org_id
 [ ] Guardrails: PROMPT_GUARD_URL + LLAMA_GUARD_URL set; GUARDRAILS_FAIL_CLOSED=true;
     Presidio installed (+ spaCy model)
 [ ] CORS_ORIGINS restricted to your front-end origins (not *)
 [ ] CAP_TESTKIT_ENABLED=false (unless deliberately used)
 [ ] Rate limiting / quotas at the gateway; PER_ORG_CONCURRENCY tuned
 [ ] Observability on: Langfuse + /metrics scraped; alert on guardrail-block spikes,
     model-endpoint errors, and DB/Qdrant latency
 [ ] Embedding model + dim pinned identical between the app and the ingestion cron
 [ ] Load test the GPU services (SGLang/TEI/guards) at expected concurrency

------------------------------------------------------------------------------
 18. BOOT VALIDATION (what the prod guard enforces)
------------------------------------------------------------------------------
 With ENVIRONMENT=prod the app refuses to start and lists every problem if:
   - DATABASE_URL is empty
   - JWT_SECRET is the dev default or < 32 bytes
   - API_KEYS is empty or still the dev default
   - GUARDRAILS_ENABLED=true but PROMPT_GUARD_URL / LLAMA_GUARD_URL are unset
     (when injection/topic checks are on)
   - DEBUG=true or SEED_DEMO_DATA=true
   - an enabled tool module (easm/brand/aci) has no MCP URL
 Fix everything it prints, then it boots. This is the "no insecure prod" guarantee.
